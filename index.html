HTML

  • Code
    • Tags
      • head
        • Contains information, or metadata, that helps the browser to render the page correcly
        • Elements included in the head are not visible when a user looks at a webpage
        • The head element in this case defines the following three elements
          • charset
            • Sets our character encoding
          • http-equiv
            • Sets an HTTP header for content
          • viewpoint
            • Sets the default view for our webpage
      • body
        • Contains content that is shown to the user
        • All the elements that we want the user to see and interact with must be placed inside this element
        • Defines the body of the document, which contains the content that is visible to the user
        • All of the content that we want to display to users must be inside this element
        • This is an example of a semantic HTML element
      • html
        • This represents the root or start of the html document
        • HTML attributes define additional information about an element and are always added to the beginning element
      • meta
        • Defines metadata such as the character set, description, keywords, author and viewpoint
        • This contains informaion used by the browser and isnt visible to the user
      • title
        • Defines the title of the webpage.
        • This element is required and is extremely important for the SEO
      • !DOCTYPE
        • This informs the browser what document type it should expect when it is opened
        • By adding HTML, we are informing the document that it should expect HTML on the page
        • This particular tag is not case-sensitive because it is a declaration
      • lang="blank"
        • Would be found inside an html tag
        • This tells the browser the language of the webpage
        • Example: adding "en" would inform the webpage that it is written in english
      • style
        • Defines the CSS styles associated with a webpage
      • base
        • Defines the base URL for a webpage
      • link
        • Connects an external resource to the HTML
      • script
        • Embeds client-side scripts in the HTML
      • div
        • Nonsemantic element
        • Creates a division in the page
      • span
        • Creates an inline division
      • p
        • Represents a paragraph or block of text
        • This element is used to wrap text on your webpage
      • ul
        • unordered lists

        • ordered lists
      • li
        • list items
      • img
        • Contains information about images that are displayed on the webpage

      • a
        • Stands for anchor
        • Creates a link to the same webpage or other webpages
      • header
        • Appears at the top of the page and usually contains a navigation menu and/or introductory content

        • Semantic element that defines a section in a document that conatins thematic content

        • Used by HTML as a way to identify elements and select them

  • Terminology
    • HTML or HyperText Markup Language
      • The most basic building block of websites. It handles the content along with each element of the webpage and its structure.
      • It is one of the three core languages of the web. (see JavaScript or CSS)
    • Elements
      • Refers to the type of content on a webpage. These elements help provide a structured framework to a webpage and make user navigation easier.
      • Common elements include (these specific elements are visible on the page):
        • Text elements
        • Image elements
        • Header elements
        • List elements
      • Other elements (these elements are specifically used to describe and divide the content on the page for the computer and other developers):
        • Meta-related elements
        • Content division elements
        • Semantic elements
      • Each HTML element performs a unique roll.
    • Metadata
      • Data about our webpage that the browser uses to effectively serve our webpage users
      • Metadata is not typicall visible to users and contains important information about accessibility, search engines, and performance
    • Child elements
      • Elements that are nested inside other elements
    • Parent elements
      • The element that holds all the children elements
    • Semantic HTML elements
      • The name of the element describes its content and the role it plays on a webpage
      • When possible, it is important to use semantic HTML elements, because they make your HTML easier to read
      • Helps to make sure the webpage is accessible to the widest variety of users, including those who use assistive technologies such as screen readers
    • Element
      • Refers to the entire element from opening tag to closing tag
    • Tag
      • Refers only to what appears inside the angle brackets
    • Heading
      • h1,h2,h3,h4,h5,h6
        • Represents the level of heading a given text block represents
      • Larger or more prominent titles and subtitles to organize a page
  • Structure
    • Elements are composed of an opening tag and closing tag represented with angle brackets (< >)
      • These tags contain everything that we want to include in the element.
      • This can include content and even other nested elements
      • Tag names are contained inside of the angle brackets
      • Closing tags begin with a forward slash /
      • Content goes between the opening tag and closing tag so that the browser understands how to treat the content
    • Declarations are not case-sensitive; HTML elements are case-sensitive
    • These specific metadata tags are found inside of the head element
      • Title
      • Style
      • Link
      • Meta
      • Script
    • Order matters
      • h1 - h6: the order of heading sizes. h1 should only appear on a page once
      • Make sure you wrap each list item to create a new bullet section
      • Assigning colors in CSS can be done with
        • semantic references
        • hexadecimal code or hex #

CSS

  • Code
    • *
      • Special Wild card symbol
      • Used to apply a rule to all the elements visible on the page
  • Structure
    • CSS Rule
      • div{color: blue: text-align: justify;}
        • Selector
          • div
        • Declaration
          • Is sequenced by a property: value; combo
          • Contains two important components
        • Property
          • Example: color or text-align
          • What we want to apply
        • Value
          • Example: blue or justify
          • The value of the applied property
        • Margin
          • Indicates how much space we want around the outside of an element
          • Is not visible on the page; it merely appears as a buff
          • 0 indicates we want NO extra space around the element
        • Padding
          • Adds space around the content inside an element
          • Makes the entire element bigger
          • Is located inside the element, whereas the margin is the transparent buffer outside an element
      • CSS header and footer selector example
        • header,
        • footer {
          • width: 100%;
          • height: 120px;
          • background-color: blue;
          • color: white;
        • }
        • We combined two rules with a comma, allowing the declarations to affect both selectors
        • This allows us to share specific styles with two or more groups and also keeps us from writing DRY codes
  • Terminology
    • CSS or Cascading Style Sheets
      • Used to describe the presentation and how each element is rendered on the webpage
      • Gives the content style by defining things like: color, aesthetics, visual layout, and organization
    • Inline CSS
      • Apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only
      • They're useful for quick and permanent changes but are less flexible than external and internal style sheets, because each inline must be edited individually whenever you make a design change
    • Internal style sheet
      • Contains CSS rules for the webpage in the head sector of the HTML file
      • The rules only apply to that page and you can configure CSS classes and IDs to style multiple elements in the page code
    • External CSS style sheet
      • The most common way to style a webpage
      • It allows developers to keep all their CSS rules in a separate file. which makes design changes easier
    • Separation of concerns
      • A widely used term in web development and refers to separating code into different files based on one
      • Based on the idea that each section of code should have its own responsibility
    • CSS box model
      • Every HTML element is essentially a box. Each box consists of margins outside the element, and the element's borders, padding, and content
    • DRY or do not repeat yourself
      • A coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained
      • This can greatly reduce the overall size and complexity of a codebase
