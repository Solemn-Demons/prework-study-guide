HTML

  • Terms
    • Elements (type of content on a webpage)
      • Text
      • Image
      • Header
      • List
      • Meta-related
      • Content division
      • Semantic
      • !DOCTYPE: is not case-sensitive because it is a declaration (declarations are not case sensitice but HTML elements are)
      • HTML attribute defines additional information about an element. (HTML attributes are always defined in the beginning element, never the closing element.)
      • Semantic HTML elements : the name of the element describes its content and the role it plays on a webpage
      • opening tag/closing tag : the content of a working HTML element is contained between
      • Element : refers to the entire element from opening tag to closing tag
      • Tag : refers only to what appears inside the angle brackets
      • heading : represents the level of given text block on the page
  • Code
    • head : contains information or metadata that helps the browser to render the page correctly. Elements within the head are not visible when a user looks at a webpage
    • body : contains the content show to the user. All the elements that we want the user to see and interact with must be place inside this element
    • !DOCTYPE html : This decleration tells the browser what document type it should expect when it is opened
    • html : an opening tag which represents the root, or start, of our HTML document
    • lang= (is the language) "(en)" : is english
    • title : elements defines the title for a webpage. (This element is required and is extremely important for Search Engine Optimization (SEO))
    • style :defines the CSS styles associated with a webpage
    • base : defines the base URL for a webpage
    • link : connects an external resource to the HTML document
    • meta : defines metadata such as the character set, description, keywords, author, and viewpoint
    • script : elements embeds client-side scripts in the HTML document
    • charset (added to) : character set
    • http-equiv (added to) : sets our character encoding
    • viewport (added to) : sets the default view for our webpage
    • div : creates a division in the page
    • span : to create an inline division
    • h1 - h6 : decides the size of the text rendered on the page
    • p : elements represents a paragraph or block of text
    • ul : unordered lists
    • ol : ordered lists
    • li : list items
    • img : contains information about images that are displayed on the webpage
    • src : attribute is the most important because it defines the location of the image
    • br : creates a line of empty space, or a line break, between two blocks of content
    • a (anchor) : creates links to the same webpage or other webpages
    • a href= : attribute point to the URL for the link
    • src : attribute defines the location, or source, of the image file
    • alt : attribute contains a text string that describes the appearance and functionality of the image. Is important for screen readers and other assistive technologies. Alt text can also improve search engine rankings for your website
    • main : element that doesn't hold any code at the moment
    • footer : appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation
    • section : are semantic elements that define a section in a document that contains thematic content
    • id : a way to identify elements and select them

Git

  • Git code
    • git status : checks what branch we are currently on
    • git checkout -b (branch) : creates a new branch
    • git : command must precede every Git statement
    • Checkout : command is to move the working branch to a new branch
    • -b : flag creates a new branch
    • add : command adds modifications in the current working branch to the staging area
    • -A : flag indicates that we want to add all changes
    • -m "message": flag to associate a message with our commit. Commit messages should be short descriptive messages of the changes implemented
    • pull : command to receive a branch's modifications into the local environment
    • origin : keyword indicates the source of the pull will be in the GitHub repo
    • push : command to push the changes we have locally to our remote GitHub branch
    • git clone (paste copied SSH value) : clones all the folders and files, but also a .git file that contains Git version control
    • git clone (paste copied SSH value) : clones all the folders and files, but also a .git file that contains Git version control
    • git pull origin main : pulls in the base branch, it is a good idea to do before each sync to make sure your code is properly up-to-date
    • git push origin : pushes all of our changes
  • SSH Keys
    • ls -al ~/.ssh: To see if any existing SSH keys are present
    • .pub: is a public SSH
    • ssh-keygen -t (name) -C "email" : creates new SSH key
    • eval "$(ssh-agent -s)" : Starts ssh-agent in terminal
    • git config : Change the name that's associate with your git commits
    • git revert (enter SHA): Will retern the code to an earlier SHA
    • blocker: being stuck on a particular problem
    • bottleneck: a point in our development process that is slowing down the rest of production
  • GitHub
    • # : Heading 1
    • ## : Heading 2
  • VS code
    • code : Opens the VS without a file
    • code . : Opens the VS with a file
  • CLI
    • ls : list of files and directories on our computer
    • cd : command allows us to change directories
    • pwd (print working directory) : will print out the directory that we are currently in to terminal
    • mkdir : create a directory
    • touch : to create a new file
    • open (~/ file) : Opens the file

Terminology

  • package: Contains code written by others that we can add to and use on our computers and in our projects
  • package manager: Like Homebrew, automates the process of installing, upgrading, configuring, and removing packages.
  • code snippet: a portion of code
  • agile software development (used by many in the coding industry): helps developers handle change and uncertainty during the development process
  • terminal: the application that allos us to interact with the shell
  • shell: the software that interprets and executes the various commands that we type in the terminal
  • bash or bash commands: used to describe the shell and the type of commands that we will use
  • pair programming: occurs when two developers collaborate on a feature or a project
  • driver-navigator model: The developer using the keyboard to type code is known as the "driver" and the "navigator" is responsible fr the overall direction that the pair decides to go
  • GitHub: a cloud-based service that allows developers to store, manage and collaborate on code
  • Git: software on your computer that tracks changes that you make to your code. It is a version control system that lets you manage and keep track of each version of your project
  • ssh-agent : is a program that keeps track of a user's identity keys and passphrases
  • web development project : is a web appliation, or web app, that provides a solution to a real-world problem
  • MVP (minimum variable product) : This is usually the first iteration of a product without extra features
  • SHA or hash : a long string of numbers or letters
  • Markdown (.md) : a lightweight plain-text formatting language
  • feature: a functional part of the application or website that has value or fulfills a need in the business
  • graphical user interface (GUI) : a way for users to interact with the computer visually
  • IDE : contains many essential tools needed for code development such as a source code editor, local build automation, a debugger and the ability to extend the IDE's capabilities with extensions to meet specific development needs
  • Branch : a copy or version of the code that can be worked on independently in isolation so development changes do not disrupt other team members who are also working on the code and testing their solutions
  • main (or master) : default or base branch
  • pull request (PR) : used to merge a comparison branch (in our case, the feature branch) into the base branch
  • CLI : how to navigate through directories and files, create a new folder for our boot camp files, and create a new file in the boot camp folder.
  • Staging area : informs Git that the changes will be added in the next commit
  • URL : a fundamental web development concept. The unique address given to a specific resource on the web
  • Hypertext Markup Language (HTML) : gives the content structure
  • Cascading Style Sheets (CSS) : give the content style
  • JavaScript (JS) : to handle logic and allow users to interact with a website
  • Child elements: elements nested inside other elements
  • Parent elements: element that holds all the children
